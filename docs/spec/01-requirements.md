# 요구사항 명세 (유저스토리 · as-built)

> SDD(Spec-Driven Development)의 1단계 문서. **실제 구현된 제품에서 역기획**한 요구사항이다.
> "PRD가 말한 것"이 아니라 "지금 동작하는 것"을 기준으로 한다(예: 검증 게이트는 제거됐고, 구인구직·콘텐츠는 구현됨).
> 형식: 유저스토리(As a / I want / so that) + 인수기준(Given/When/Then). 상위 기획 맥락은 [PRD](../hallym-mice-alumni-platform-prd.md), 결정 근거는 [DECISIONS](../DECISIONS.md).

## 페르소나

| # | 페르소나 | 목표 | 권한 |
|---|---|---|---|
| P1 | **재학생** | 선배의 경험·기회를 얻는다 | member |
| P2 | **현직 동문** | 소속을 노출하고 동문과 연결된다 | member |
| P3 | **시니어/대표** | 인재 탐색·채용·홍보 | member |
| P4 | **외부 파트너** | 협업(이름·회사·직무만 열람, 오픈카톡 비열람) | member(role=partner) |
| P5 | **운영진** | DB 유지·보호·모더레이션 | admin |

## 비기능 요구사항 (NFR)

- **NFR-1 보안**: 전 테이블 RLS deny-all + service_role 단일경로 + 전 핸들러 `withAuth`. 사용자 입력으로 `role/status/is_verified` 변경 불가.
- **NFR-2 프라이버시(PIPA)**: 분리 동의 3종 수집·기록(`consents`), 탈퇴 시 식별정보 익명화/파기, 항목별 공개 토글, 타인 이메일·학번·오픈카톡 마스킹.
- **NFR-3 모바일 우선**: 기준 375px, 터치 타깃 44px+, 확대 허용.
- **NFR-4 접근성**: 텍스트 대비 4.5:1+, 입력 라벨 연결, 이미지 alt, `prefers-reduced-motion` 존중.
- **NFR-5 무료 인프라**: Vercel Hobby + Supabase 무료 + R2 10GB + Resend 무료 한도. 벤더 비종속(이식성).
- **NFR-6 인수인계 단순성**: 무의존성 우선, 문자열 중앙화(`lib/messages`·`lib/labels`), 한 패턴을 복제해 학습 비용 최소화.
- **NFR-7 성능**: 업로드 이미지 클라이언트 압축(WebP), 리스트 lazy 로딩, 무한 스크롤.

## 에픽 & 유저스토리

### E1. 가입 · 온보딩
- **US-1.1** As a 방문자, I want 구글 계정으로 가입, so that 비밀번호 없이 30초에 시작.
  - Given 비로그인, When "시작하기"→구글 로그인 성공 & 프로필 없음, Then `/onboarding`으로 이동.
- **US-1.2** As a 신규 가입자, I want 최소 정보(이름·역할·학과·졸업연도 또는 학번) + 동의 3체크로 즉시 회원, so that 게이트 없이 바로 사용.
  - Given 온보딩 폼, When 필수 + 동의 3개 충족 후 제출, Then `status=active` 회원 생성 & 홈으로(검증 대기 없음).
  - Given 동의 미체크, When 제출, Then 제출 비활성/거부.
- **US-1.3** As a 운영자, I want 만 14세 미만 가입 차단 고지, so that 법적 요건 충족.

### E2. 동문 디렉토리 · 검색
- **US-2.1** As a 회원, I want 회사·직무·분야 태그·기수·커피챗 가능 여부로 검색·필터, so that 원하는 동문을 찾는다.
  - Given 디렉토리, When 키워드/필터 적용, Then 매칭 카드 목록(무한 스크롤). 검색은 이름·회사·직무 부분일치.
  - Given "커피챗 가능" 필터, When 적용, Then `open`/`monthly`만(제안만=offer_only 제외).
- **US-2.2** As a 회원, I want 카드에서 오픈카톡 URL이 보이지 않게, so that 연락처가 무차별 수집되지 않는다. (상세에서만 정책 통과 시 노출)
- **US-2.3** As a 비로그인 사용자, I want 디렉토리에 접근 불가, so that 실명 정보가 외부에 노출되지 않는다.

### E3. 프로필 · 공개 범위
- **US-3.1** As a 회원, I want 회사·직무·소개·경력·태그·사진을 채우고 항목별 공개/비공개 토글, so that 노출 범위를 스스로 통제.
  - Given 내 정보 편집, When 필드별 토글 변경 후 저장, Then 타인에게 비공개 항목 숨김(이름·역할·커피챗·태그는 항상 공개).
- **US-3.2** As a 회원, I want 프로필 사진 등록·변경·삭제가 **즉시 적용**, so that 별도 저장 없이 반영. (업로드는 자동 압축)
- **US-3.3** As a 회원, I want 프로필을 비공개로 전환(데이터 유지), so that 잠시 쉴 때 디렉토리에서 빠진다.

### E4. 연락 (오픈카톡 · 이메일 제안)
- **US-4.1** As a 회원, I want 상대가 공개한 오픈카톡으로 바로 연결, so that 부담 없이 대화 시작. (클릭 시 `coffeechat_click` 기록)
- **US-4.2** As a 회원, I want 오픈카톡이 없으면 이메일 제안을 서버 중계로 전송, so that 실제 이메일 노출 없이 연락.
  - Given 제안 허용 대상, When 메시지 전송, Then 서버가 Resend로 중계(양측 실 이메일 비노출) + 수신자에게 인앱 알림. 1일 5건 제한.
- **US-4.3** As a 외부 파트너, I want 오픈카톡은 못 보되 제안은 가능, so that 차등 접근.

### E5. 안전 (신고 · 차단)
- **US-5.1** As a 회원, I want 부적절한 프로필/공고/콘텐츠를 신고, so that 운영자가 검토. (1일 10건, 동일 대상 1일 1건)
- **US-5.2** As a 회원, I want 특정 회원을 차단/해제, so that 서로 노출·연락 차단. (해제는 내 정보에서)
  - Given 차단, When 성공, Then 양방향으로 디렉토리/상세/제안 차단 + 상세 화면 즉시 갱신.

### E6. 구인구직 (기회 · P2)
- **US-6.1** As a 회원, I want 공고를 작성, so that 채용·공모·프로젝트를 알린다.
  - Given 작성·제출, When 성공, Then `status=pending`(서버 강제) — 관리자 승인 전 보드 미노출.
- **US-6.2** As a 회원, I want 게시중 공고를 검색·필터(유형·태그)·북마크, so that 관심 공고를 모은다.
- **US-6.3** As a 회원, I want **내 공고** 목록에서 전 상태 확인·수정·마감, so that 올린 공고를 잃지 않는다.
- **US-6.4** As a 운영자, I want 대기 공고를 승인(게시)/숨김, so that 품질 관리. (승인 시 작성자에게 인앱 알림)
- **US-6.5** 마감일이 지나면 자동 마감 처리(지원 비활성).

### E7. 콘텐츠 (동문 이야기 · P3)
- **US-7.1** As a 운영자, I want **마크다운 블로그 에디터**(툴바·본문 이미지 첨부·미리보기)로 인터뷰·소식 작성, so that 풍부한 글 게시. (생성·수정 동일 풀 에디터)
- **US-7.2** As a 운영자, I want 커버 이미지·관련 동문·태그를 붙이고 게시/임시저장/숨김 전환, so that 발행 통제.
- **US-7.3** As a 회원, I want 게시된 콘텐츠를 마크다운 렌더로 읽고 관련 동문 프로필로 이동, so that 맥락 연결. (열람 시 `article_view` 기록)

### E8. 갤러리 (행사 기록 · P1.5)
- **US-8.1** As a 운영자, I want 앨범 생성·이미지 드래그&드롭 업로드(다중 진행률·자동 압축)·삭제, so that 행사 기록 큐레이션.
- **US-8.2** As a 운영자, I want **게시 동의 확인** 없이는 공개 불가, so that 초상권 보호.
- **US-8.3** As a 회원, I want 공개 앨범을 보고 라이트박스(좌우·키보드·스와이프)로 사진 열람, so that 행사 추억. (로그인 회원만)

### E9. 알림
- **US-9.1** As a 회원, I want 인박스와 헤더 벨로 알림을 받고 클릭 시 해당 화면으로 이동, so that 활동을 놓치지 않는다. (제안 도착·공고 게시 등에서 인앱 알림 생성)
- **US-9.2** 읽음 처리 시 벨 미읽음 배지 갱신.

### E10. 계정 · 탈퇴
- **US-10.1** As a 회원, I want 로그아웃을 어디서나 1탭(헤더 계정 메뉴)에서, so that 발견성.
- **US-10.2** As a 회원, I want 탈퇴 시 식별정보 익명화/파기(복구 불가), so that PIPA 파기권 행사. (2단계 확인)

### E11. 관리자 · 모더레이션
- **US-11.1** As a 운영자, I want 대시보드에서 오늘 할 일(미처리 신고)·최근 가입·빠른 링크, so that 운영 동선.
- **US-11.2** As a 운영자, I want 신고 큐(상태머신) + 대상 제목·보기 링크 + 숨김/정지, so that 무엇을 처리하는지 알고 조치.
- **US-11.3** As a 운영자, I want 회원 역할/상태/인증 배지 변경(본인 강등은 차단), so that 회원 관리. (모든 변경 `admin_logs` 기록)
